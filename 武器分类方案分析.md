# RimWorld 游戏中武器近战/远程分类方案分析

## 概述
RimWorld游戏中区分武器属于近战还是远程，采用了多层次的分类方案。这些方案既用于XML定义，也用于代码逻辑判断。

---

## 一、XML定义层面的分类方案

### 1. **weaponClasses** 属性（最核心的分类方式）
在 `ThingDef` 中定义，直接指定武器所属的类别。

#### 定义文件
- 位置：`references/RimWorld-XMLs/Core/Defs/WeaponClassDefs/WeaponClassDefs.xml`

#### 近战武器类别
```xml
<WeaponClassDef>
  <defName>Melee</defName>
  <label>melee</label>
</WeaponClassDef>

<WeaponClassDef>
  <defName>MeleePiercer</defName>
  <label>melee piercer</label>
</WeaponClassDef>

<WeaponClassDef>
  <defName>MeleeBlunt</defName>
  <label>melee blunt</label>
</WeaponClassDef>

<WeaponClassDef>
  <defName>Neolithic</defName>
  <label>neolithic</label>
</WeaponClassDef>
```

#### 远程武器类别
```xml
<WeaponClassDef>
  <defName>Ranged</defName>
  <label>ranged</label>
</WeaponClassDef>

<WeaponClassDef>
  <defName>RangedLight</defName>
  <label>ranged light</label>
</WeaponClassDef>

<WeaponClassDef>
  <defName>RangedHeavy</defName>
  <label>ranged heavy</label>
</WeaponClassDef>

<WeaponClassDef>
  <defName>LongShots</defName>
  <label>long shots</label>
</WeaponClassDef>

<WeaponClassDef>
  <defName>ShortShots</defName>
  <label>short shots</label>
</WeaponClassDef>

<WeaponClassDef>
  <defName>Ultratech</defName>
  <label>ultratech</label>
</WeaponClassDef>
```

#### 使用示例

**近战武器示例（Mace）：**
```xml
<ThingDef ParentName="BaseMeleeWeapon_Blunt_Quality">
  <defName>MeleeWeapon_Mace</defName>
  <label>mace</label>
  <weaponClasses>
    <li>Melee</li>
  </weaponClasses>
  <!-- ... -->
</ThingDef>
```

**远程武器示例（Revolver）：**
```xml
<ThingDef ParentName="BaseHumanMakeableGun">
  <defName>Gun_Revolver</defName>
  <label>revolver</label>

    <li>RangedLight</li>

```

**新石器时代弓箭：**
```xml

  <defName>Bow_Short</defName>

  <weaponClasses>
    <li>Ranged</li>
    <li>RangedLight</li>
  </weaponClasses>
  <!-- ... -->
</ThingDef>
```

### 2. **thingCategories** 属性（分类系统）
用于游戏UI和物品系统的分类。

#### 近战武器
```xml
<thingCategories>
  <li>WeaponsMelee</li>
</thingCategories>
```

#### 远程武器
```xml
<thingCategories>
  <li>WeaponsRanged</li>
</thingCategories>
```

### 3. **weaponTags** 属性（灵活的标签系统）
用于更细粒度的武器分类，支持多个标签。

#### 近战示例
```xml
<weaponTags>
  <li>MedievalMeleeDecent</li>
</weaponTags>
```

#### 远程示例
```xml
<weaponTags>
  <li>SimpleGun</li>
  <li>Revolver</li>
</weaponTags>
```

### 4. **Verb定义**（功能性区分）
通过武器的"动词"（Verb）类型区分近战和远程。

#### 近战武器：使用 `tools` 属性
```xml
<tools>
  <li>
    <label>blade</label>
    <capacities>
      <li>Cut</li>
    </capacities>
    <power>12</power>
    <cooldownTime>1.5</cooldownTime>
  </li>
</tools>
```

**Capacity（能力）类型：**
- `Cut` - 切割伤害
- `Blunt` - 钝击伤害
- `Poke` - 戳刺伤害

#### 远程武器：使用 `verbs` 属性
```xml
<verbs>
  <li>
    <verbClass>Verb_Shoot</verbClass>
    <hasStandardCommand>true</hasStandardCommand>
    <defaultProjectile>Bullet_Revolver</defaultProjectile>
    <warmupTime>0.3</warmupTime>
    <range>25.9</range>
    <soundCast>Shot_Revolver</soundCast>
  </li>
</verbs>
```

**关键区别：**
- `Verb_Shoot` - 射击动词（远程）
- `range` 属性 - 远程武器必须定义射程
- `defaultProjectile` - 远程武器必须定义发射物

### 5. **ParentName 继承链**（结构层次）

#### 近战武器继承链
```
BaseWeapon
  ↓
BaseMeleeWeapon
  ↓
BaseMeleeWeapon_Blunt（钝击）或 BaseMeleeWeapon_Sharp（锐利）
  ↓
BaseMeleeWeapon_Blunt_Quality（具有品质系统）
  ↓
具体武器定义（MeleeWeapon_Mace等）
```

#### 远程武器继承链
```
BaseWeapon
  ↓
BaseGun
  ↓
BaseGunWithQuality
  ↓
BaseMakeableGun 或 BaseHumanMakeableGun
  ↓
具体武器定义（Gun_Revolver等）
```

**基础定义中的weaponClasses指定：**

```xml
<!-- BaseGun 自动标记为远程 -->
<ThingDef Name="BaseGun" Abstract="True" ParentName="BaseWeapon">
  <weaponClasses>
    <li>Ranged</li>
  </weaponClasses>
  <thingCategories>
    <li>WeaponsRanged</li>
  </thingCategories>
</ThingDef>

<!-- BaseMeleeWeapon 自动标记为近战 -->
<ThingDef Name="BaseMeleeWeapon" Abstract="True" ParentName="BaseWeapon">
  <weaponClasses>
    <li>Melee</li>
  </weaponClasses>
  <thingCategories>
    <li>WeaponsMelee</li>
  </thingCategories>
</ThingDef>
```

## 二、代码层面的分类方案

### 1. **IsRangedWeapon / IsMeleeWeapon 属性**
RimWorld API 提供的属性，用于在C#代码中判断武器类型。

#### 使用示例
```csharp
// 项目中的实际使用
if (sourceWeapon.def.IsRangedWeapon && targetDef != null)
{
    // 处理远程武器转换逻辑
}

if (ingredient?.def?.IsRangedWeapon == true)
{
    return ingredient; // 优先返回远程武器
}

// 在条件Hediff中的使用
if (string.Equals(normalizedTag, "Ranged", StringComparison.OrdinalIgnoreCase)
    && def.IsRangedWeapon)
{
    return true;
}

if (string.Equals(normalizedTag, "Melee", StringComparison.OrdinalIgnoreCase)
    && def.IsMeleeWeapon)
{
    return true;
}
```

### 2. **Verb 类型检查**
```csharp
if (verb.IsMeleeAttack || !IsRangedWeaponOrTurretVerb(verb))
{
    return false; // 近战攻击或非远程动词
}

// 判断是否为远程武器或炮塔动词
private static bool IsRangedWeaponOrTurretVerb(Verb verb)
{
    if (verb.EquipmentSource != null || verb.ReloadableCompSource != null
        || verb.VerbOwner_ChargedCompSource != null)
    {
        return true;
    }
    return verb.caster is Building_Turret;
}
```

### 3. **weaponTags 检查**
```csharp
// 在项目的条件Hediff系统中
if (def.weaponTags?.Contains(normalizedTag) == true)
{
    return true; // 武器包含特定标签
}
```

## 三、EasyRim 项目中的应用

### 项目中的条件Hediff系统
项目在 `HediffComp_ConditionalHediffDistributor.cs` 中实现了条件Hediff系统，支持基于武器类型的条件判断：

```csharp
public List<ThingDef> allEquipment;
public List<ThingDef> anyEquipment;
public List<ThingDef> noEquipment;
public List<string> allEquipmentTags;
public List<string> anyEquipmentTags;
public List<string> noEquipmentTags;

// 支持语义别名
if (string.Equals(normalizedTag, "Ranged", StringComparison.OrdinalIgnoreCase)
    && def.IsRangedWeapon)
{
    return true;
}

if (string.Equals(normalizedTag, "Melee", StringComparison.OrdinalIgnoreCase)
    && def.IsMeleeWeapon)
{
    return true;
}
```

#### XML 应用示例（hediffs_transcendence.xml）
```xml
<ConditionalHediffRule>
  <key>transcendence-ranged-calibration</key>
  <activeHediff>EM_NexusSpine_RangedAdaptation</activeHediff>
  <anyEquipmentTags>
    <li>Ranged</li>
  </anyEquipmentTags>
</ConditionalHediffRule>

<ConditionalHediffRule>
  <key>transcendence-gene-strong-melee</key>
  <activeHediff>EM_NexusSpine_StrongMelee</activeHediff>
  <anyEquipmentTags>
    <li>Melee</li>
  </anyEquipmentTags>
</ConditionalHediffRule>
```

## 四、总体架构总结

```
分类维度
├── weaponClasses（最权威的分类标准）
│   ├── 近战类：Melee, MeleePiercer, MeleeBlunt, Neolithic
│   └── 远程类：Ranged, RangedLight, RangedHeavy, LongShots, ShortShots, Ultratech
│
├── thingCategories（UI和系统分类）
│   ├── WeaponsMelee
│   └── WeaponsRanged
│
├── weaponTags（灵活的标签系统）
│   ├── 针对具体武器类型的自定义标签
│   └── 支持"Ranged"、"Melee"语义别名
│
├── Verb 定义（功能性定义）
│   ├── 近战：tools + capacity
│   └── 远程：verbs + Verb_Shoot + range + projectile
│
└── API 属性（代码层判断）
    ├── IsRangedWeapon
    ├── IsMeleeWeapon
    └── Verb.IsMeleeAttack
```

## 五、实际应用场景

### 场景1：检查角色是否装备远程武器
```xml
<!-- Hediff 条件定义 -->
<anyEquipmentTags>
  <li>Ranged</li>
</anyEquipmentTags>
```

### 场景2：食谱转换系统
```csharp
// 仅远程武器可转换为独特武器
if (sourceWeapon.def.IsRangedWeapon && targetDef != null)
{
    product = CreateConvertedWeapon(targetDef, sourceWeapon, ...);
}
```

### 场景3：烟雾影响远程射击
```csharp
// 只有远程武器射击会受烟雾影响
if (verb.IsMeleeAttack || !IsRangedWeaponOrTurretVerb(verb))
{
    return false;
}
```

## 六、关键属性对照表

| 属性 | 近战武器 | 远程武器 | 说明 |
|------|---------|---------|------|
| **weaponClasses** | Melee, MeleePiercer, MeleeBlunt | Ranged, RangedLight, RangedHeavy | 权威分类 |
| **thingCategories** | WeaponsMelee | WeaponsRanged | 系统分类 |
| **tools** | ✓ 必需 | ✗ 不使用 | 定义攻击能力 |
| **verbs** | 可选 | ✓ 必需（Verb_Shoot） | 定义动作 |
| **range** | ✗ | ✓ 必需 | 射程 |
| **defaultProjectile** | ✗ | ✓ 必需 | 发射物 |
| **capacity** | Cut, Blunt, Poke | ✗ | 伤害类型 |
| **IsRangedWeapon** | false | true | 代码判断 |
| **IsMeleeWeapon** | true | false | 代码判断 |
